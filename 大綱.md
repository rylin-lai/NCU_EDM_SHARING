## Slide 9: pytest

- The industry-standard testing framework featuring fixtures for setup/teardown, parametrize for data-driven tests, and markers for test categorization and selective execution
- pytest-xdist
- Parallel test execution across multiple CPU cores or machines, dramatically reducing test suite runtime for large codebases
- pytest-html
- Generates beautiful, shareable HTML test reports with detailed failure information, perfect for CI/CD pipelines and stakeholder communication
- coverage
- Measures code coverage percentage, identifies untested code paths, and generates detailed reports to improve test completeness
- GitHub Actions
- Cloud-based CI/CD automation for build, test, and deployment workflows with matrix testing across multiple platforms and Python versions
- Enterprise Strategy: Use Python to test all languages (Go, C, Python, JS). Testing workflows become versioned, repeatable, and fully automated—the foundation of reliable software delivery.

## Slide 13: Automating Your Workflow

- The optional third lab takes your testing to the next level by integrating with GitHub Actions. You'll experience how Python tests become part of an automated CI/CD pipeline that runs on every code change.
- What You'll Build
- GitHub Actions workflow YAML configuration
- Multi-platform test matrix (Linux, Windows, macOS)
- Automated pytest execution
- HTML report generation and artifact upload
- Build status badges for your repository
- Workflow Configuration
- name: Test Suiteon: [push, pull_request]jobs:  test:    strategy:      matrix:        os: [ubuntu-latest, windows-latest]        python: [3.9, 3.10, 3.11]    runs-on: ${{ matrix.os }}    steps:      - uses: actions/checkout@v3      - uses: actions/setup-python@v4      - run: pip install pytest pytest-html      - run: pytest --html=report.html      - uses: actions/upload-artifact@v3
- This configuration tests your code across multiple operating systems and Python versions automatically.
